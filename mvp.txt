## 1. MVP Goal

Build a web-based store management system for a small/medium electronics shop.

The MVP should cover the most important daily operations:

1. Login and role-based access.
2. Product management.
3. Customer management.
4. Inventory tracking.
5. Import/export stock.
6. POS sales and invoice creation.
7. Basic reports: revenue, top products, low-stock alerts.

Leave advanced features such as full supplier purchase order workflow, customer pre-order blocking, PDF/Excel export, and complex promotion rules for V1.1.

## 7. Backend Wiring Example

### Route → Controller → Service → Repository

Example: create sale.

```js
// routes/invoice.routes.js
const express = require("express");
const router = express.Router();
const InvoiceController = require("../controllers/InvoiceController");
const auth = require("../middlewares/auth.middleware");
const allowRoles = require("../middlewares/role.middleware");

router.post(
  "/sale",
  auth,
  allowRoles("manager", "salesperson"),
  InvoiceController.createSale
);

module.exports = router;
```

```js
// controllers/InvoiceController.js
const InvoiceService = require("../services/InvoiceService");

exports.createSale = async (req, res, next) => {
  try {
    const result = await InvoiceService.createSale({
      ...req.body,
      employee_id: req.user.id
    });

    res.status(201).json({
      message: "Sale completed successfully",
      data: result
    });
  } catch (error) {
    next(error);
  }
};
```

```js
// services/InvoiceService.js
const db = require("../config/db");
const InvoiceRepository = require("../repositories/InvoiceRepository");
const InventoryRepository = require("../repositories/InventoryRepository");
const CustomerRepository = require("../repositories/CustomerRepository");
const { generateInvoiceCode } = require("../utils/generateCode");

exports.createSale = async (payload) => {
  const client = await db.connect();

  try {
    await client.query("BEGIN");

    if (!payload.items || payload.items.length === 0) {
      throw new Error("Cart is empty");
    }

    for (const item of payload.items) {
      const inventory = await InventoryRepository.findByProductIdForUpdate(
        client,
        item.product_id
      );

      if (!inventory || inventory.quantity < item.quantity) {
        throw new Error(`Not enough stock for product ID ${item.product_id}`);
      }
    }

    const subtotal = payload.items.reduce((sum, item) => {
      return sum + Number(item.unit_price) * Number(item.quantity);
    }, 0);

    const discountAmount =
      payload.discount_type === "percent"
        ? subtotal * Number(payload.discount_value || 0) / 100
        : Number(payload.discount_value || 0);

    const totalAmount = subtotal - discountAmount;
    const pointsEarned = Math.floor(totalAmount / 10000);

    const invoice = await InvoiceRepository.createInvoice(client, {
      invoice_code: generateInvoiceCode("HD"),
      transaction_type: "sale",
      customer_id: payload.customer_id || null,
      employee_id: payload.employee_id,
      discount_type: payload.discount_type || null,
      discount_value: payload.discount_value || 0,
      subtotal,
      total_amount: totalAmount,
      payment_method: payload.payment_method,
      status: "completed",
      points_earned: pointsEarned
    });

    for (const item of payload.items) {
      await InvoiceRepository.createInvoiceItem(client, {
        invoice_id: invoice.id,
        product_id: item.product_id,
        quantity: item.quantity,
        unit_price: item.unit_price
      });
    }

    if (payload.customer_id) {
      await CustomerRepository.addPoints(client, payload.customer_id, pointsEarned);
    }

    await client.query("COMMIT");

    return {
      invoice,
      subtotal,
      discount_amount: discountAmount,
      total_amount: totalAmount,
      change_amount: Number(payload.amount_received || 0) - totalAmount
    };
  } catch (error) {
    await client.query("ROLLBACK");
    throw error;
  } finally {
    client.release();
  }
};
```

## 9. MVP Database Notes

Use the SQL schema from the document as the starting point.

For safer MVP behavior, add these constraints:

```sql
ALTER TABLE products
ADD CONSTRAINT chk_product_prices
CHECK (cost_price >= 0 AND selling_price >= 0);

ALTER TABLE inventory
ADD CONSTRAINT chk_inventory_non_negative
CHECK (quantity >= 0);

ALTER TABLE invoices
ADD CONSTRAINT chk_invoice_amounts
CHECK (subtotal >= 0 AND total_amount >= 0);

ALTER TABLE products
ADD CONSTRAINT chk_min_stock_non_negative
CHECK (min_stock_qty >= 0);
```

Also, before inserting `invoice_items` for a sale, the backend should lock inventory rows using:

```sql
SELECT *
FROM inventory
WHERE product_id = $1
FOR UPDATE;
```

This prevents two cashiers from selling the last unit at the same time.

## 11. MVP Acceptance Criteria

The MVP is complete when:

```txt
A manager can log in and view all modules.
A salesperson can log in and only use POS, products, customers, inventory, invoices.
A warehouse staff can log in and use inventory and warehouse import/export.
A product can be created, edited, searched, and soft-deleted.
A customer can be created and searched by phone.
Stock import increases inventory.
Stock export decreases inventory and blocks export if stock is insufficient.
POS sale creates invoice and invoice items.
POS sale automatically decreases inventory.
Customer points increase after sale.
Low-stock products are highlighted.
Manager can view revenue and top product reports.
Unauthorized users cannot access restricted APIs or pages.
```
